# 컴포넌트 기능 변형 심층 분석

[UI 컴포넌트 총망라 체크리스트](./UI_컴포넌트_총망라_체크리스트.md) 18번 섹션에서 Select 하나로 시연했던 "같은 컴포넌트도 기능 범위가 다르다"는 분석을, 실제로 먼저 코드화될 티어1~2 핵심 컴포넌트 + MES 특화 컴포넌트로 확장한 문서다. 일반 SaaS 관례와 MES 도메인 특성을 함께 고려해, 각 컴포넌트마다 "우리가 실제로 필요한 사양"을 결론으로 못박는다.

**사용법**: `결론` 행이 실제 구현 스펙이다. 나머지 변형은 참고용(향후 확장 여지)이며 지금 당장 만들 필요는 없다.

## 표시·컨테인먼트류

### Table / Data Grid (가장 중요 — MES 핵심)

| 변형 | 설명 | SaaS 일반 사례 | MES 필요성 |
|---|---|---|---|
| 정렬(Sort) | 컬럼 헤더 클릭 정렬 | 거의 모든 데이터 그리드 기본 | 필수 |
| 필터(Filter) | 컬럼별/전체 필터 | 대부분 지원 | 필수 (품목·설비·기간 필터 흔함) |
| 컬럼 고정(Frozen Column) | 가로 스크롤 시 특정 컬럼 고정 | 대용량 그리드에서만 | 필수 (품목코드·설비명 등 식별 컬럼 고정) |
| 다단 헤더(Grouped Header) | 2단 이상 헤더 | 재무·통계형 그리드 | 필수 (예: 상한/평균/하한을 한 그룹으로 묶는 헤더) |
| 행 그룹화/펼치기 | 그룹 단위 집계 + 하위 행 펼치기 | 재무·리포트형 | 필요 (라인별→설비별 드릴다운) |
| 중첩 그리드(Master-Detail) | 행 클릭 시 하위 그리드 표시 | 주문-상세 같은 관계형 데이터 | 필요 (작업지시-실적 관계) |
| 셀 인라인 편집 | 셀 클릭 후 바로 수정 | 스프레드시트형 UI | 이미 있음(확인됨) |
| 행 다중 선택 + 일괄 처리 | 체크박스 선택 후 툴바 노출 | 관리자 도구 공통 | 이미 있음(부분 확인) |
| 가상 스크롤 | 수천 행 이상에서 성능 최적화 | 대용량 시 필수 | 필요 (설비 로그·이력 조회 화면) |
| 셀 상태 색상 | 값에 따라 배경/텍스트 색 변경 | 대시보드형 | 필수 (합격/불합격, 정상/경고 색상) |
| 합계/집계 행(Footer Row) | 하단에 합계·평균 고정 행 | 회계·재고 그리드 | 필요 (수량 합계, 평균값 등) |
| 컬럼 리사이즈/순서변경/숨김 | 사용자가 그리드 커스터마이징 | 관리자 도구 공통 | 필요하나 우선순위는 낮음 |
| Export(엑셀/인쇄) | 그리드 데이터 내보내기 | 거의 모든 업무 시스템 | 이미 있음(확인됨) |

**결론**: 정렬·필터·컬럼고정·다단헤더·행그룹화·중첩그리드·가상스크롤·셀상태색상·합계행 9가지가 실제 구현 스펙. TanStack Table(헤드리스) + 기존 Table 시각 스타일 조합을 권장 — 헤드리스 라이브러리가 정렬/필터/가상스크롤/그룹화 로직을 이미 제공하므로 새로 개발할 필요 없이 스타일만 입히면 된다.

### Badge

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 텍스트 배지 | "완료", "대기" 등 텍스트 | 이미 있음 |
| 숫자 배지(카운트) | 알림 개수 등 | 필요 (미확인 알람 개수 등) |
| 점 배지(Dot) | 숫자 없이 점만 | 필요 (헤더 알림 아이콘용) |
| 아웃라인/솔리드 | 배경 채움 여부 | 이미 있음(3색+제거가능형) |

**결론**: 기존 텍스트 배지에 숫자 배지 + 점 배지 2종만 추가하면 충분.

### Pagination

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 숫자 페이지네이션 | 1 2 3 ... 페이지 이동 | 필수 (조회 화면 표준) |
| 무한 스크롤 | 스크롤하며 자동 로드 | 모바일 리스트에 적합 |
| 더보기 버튼 | 클릭 시 다음 페이지 추가 로드 | 대안 |

**결론**: 데스크탑은 숫자 페이지네이션, 모바일은 무한 스크롤 또는 더보기 버튼 — 데이터 조회 화면(테이블) 특성상 숫자형이 메인이고 모바일 리스트류만 무한 스크롤 적용.

## 입력류

### Combobox / Multi-select

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 정적 필터형 | 이미 로드된 옵션 내 검색 | 필수 (품목·설비 목록이 크지 않을 때) |
| 비동기 검색형 | 입력할 때마다 서버 재조회 | 필요 (품목·거래처 수가 많을 때) |
| 다중 선택 + 태그 표시 | 선택값을 칩으로 표시 | 필요 (여러 설비·라인 동시 필터) |
| 신규값 생성(Creatable) | 목록에 없는 값 즉시 추가 | 불필요 (MES는 마스터 데이터 기반이라 임의 생성 부적절) |

**결론**: 정적 필터형 + 비동기 검색형 + 다중 선택 3가지 조합 지원. Creatable은 제외.

### Date Picker / Date Range Picker

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 단일 날짜 선택 | 달력에서 하루 선택 | 필수 |
| 기간(범위) 선택 | 시작~종료 선택 | 필수 (조회 기간 필터 표준) |
| 날짜+시간 선택 | 시분초까지 | 필요 (설비 로그·알람 조회) |
| 특정일 비활성화 | 주말·공휴일 등 선택 불가 처리 | 필요 (생산일자 등록 시 휴무일 제외) |
| 빠른 선택 프리셋 | "오늘", "최근 7일" 등 버튼 | 필요 (조회 화면 UX 표준) |

**결론**: 단일/범위/날짜+시간 3종 모두 필요, 프리셋 버튼 포함. 특정일 비활성화는 해당 화면(생산계획 등록류)에만 선택 적용.

### Checkbox

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 단일 체크 | 기본형 | 필수 |
| 전체선택(Indeterminate) | 하위 일부만 선택 시 반선택 표시 | 필수 (테이블 전체선택 체크박스에 이미 필요한 패턴) |

**결론**: 단일 + indeterminate 2가지. Tri-state(체크/해제/제외) 같은 3단계는 불필요.

## 오버레이류

### Toast / Snackbar

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 자동 소멸형 | 몇 초 후 자동 사라짐 | 필수 (저장 완료 등 단순 알림) |

**결론**: 자동 소멸형 기본 + 액션 포함형(삭제 취소 등 파괴적 액션에만) 2종.

| 변형 | 설명 | MES 필요성 |
|---|---|---|
| 단순 텍스트형 | 호버 시 짧은 설명 | 필수 |
| 리치 콘텐츠형 | 이미지·서식 포함 | 불필요 |

**결론**: 단순 텍스트형만 필요.

## MES 특화 재검토

### 검사 버튼 (○/△/✕/-)

**결론(확정)**: 실제로는 ○/△/✕/- 4상태. 4개 버튼 모두 컴포넌트에 포함해서 설계·구현한다. "보류"라는 별도의 5번째 상태는 이번 스코프에서 제외(요구사항 미확정).

**"-"(대시)의 정확한 의미는 미확정**: 버튼 자체는 4개 중 하나로 이미 존재하지만, 이 "-"가 정확히 무엇을 뜻하는지가 아직 불명확 — 팀 내부적으로는 보류의 의미로 넣었는데 실제 사용 현장에서는 다르게 쓰이고 있는 것으로 보임(예: 해당없음/미검사 등 다른 의미일 가능성). 사용자 쪽에서 내부 확인 후 알려주기로 함. **버튼 UI(4개 중 하나로 존재)는 지금 그대로 구현하되, 그 의미에 따라 달라질 수 있는 부분(라벨, 색상, 클릭 시 동작 등)은 의미가 확정되면 반영.**

### 그리드 세부 패턴 (14번 섹션) 변형 재확인

이미 13개 패턴이 목록화돼 있고, 위 Table/Data Grid 분석에서 우선순위가 재확인됐다. 추가로 고려할 변형:
- **합계/집계 행**: 기존 13개 목록에 없던 항목 — 새로 추가 권장 (Tier 3)
- **컬럼별 정렬 방향 표시(화살표 아이콘)**: 기존 아이콘 세트(icon-arrow류) 재사용 가능, 별도 디자인 불필요
